# 이웃콜 비개발자 소개서

개발 지식이 없어도 이웃콜이 무엇인지, 왜 필요한지, 어떻게 체험하면 되는지 이해할 수 있도록 만든 소개서입니다.

---

## 이웃콜이 뭔가요?

쉽게 말하면 이렇습니다.

> 카카오T나 쏘카 같은 배차 앱을 직접 만들어서 수수료 없이 운영할 수 있게 해 주는 도구입니다.

택시, 대리운전, 화물, 퀵서비스, 농어촌 셔틀처럼 서로 다른 이동 서비스를 하나의 시스템으로 연결합니다.
"고객이 호출하고, 기사가 수락하고, 운영자가 전체 상태를 보는 구조"를 직접 운영할 수 있게 해 주는 기반 도구입니다.

---

## 왜 이런 걸 만들었나요?

지금 많은 기사와 라이더들은 플랫폼 앱에 의존해 일을 받습니다.
문제는 그 플랫폼이 중간에서 높은 수수료를 가져간다는 점입니다.
기사는 플랫폼 없이는 손님을 받기 어렵고, 플랫폼은 그 상황을 유지하려 합니다.

이웃콜은 이 구조에 의문을 품습니다.

> "배차 시스템 자체를 누구나 쓸 수 있게 공개하면, 지역 단위 서비스나 기사 조합도 직접 운영할 수 있지 않을까?"

그래서 이웃콜은 **무료(오픈소스)** 입니다.
코드를 내려받아 자기 서버에 올리면, 수수료 없이 직접 배차 플랫폼을 운영할 수 있습니다.

---

## 누구에게 도움이 되나요?

| 대상 | 어떻게 쓸 수 있나요? |
|------|------------------|
| 지역 택시·퀵 네트워크 | 자체 배차 앱 운영 |
| 기사 협동조합 | 수수료 없는 조합 운영 플랫폼 |
| 농어촌 교통 실험 | 호출형 셔틀 운영 기반 |
| 사회적 기업·공공기관 | 공공 이동 서비스 플랫폼 |
| 개발 공부 중인 사람 | 한국형 배차 시스템 구조 학습 |

---

## 어떤 서비스를 생각하고 있나요?

이웃콜은 하나의 앱만 만드는 프로젝트가 아닙니다.
여러 이동 서비스를 하나의 공통 시스템으로 묶는 구조입니다.

| 모듈 | 서비스 |
|------|--------|
| FREECAB | 택시 호출 |
| FREEDRIVE | 대리운전 |
| FREECARGO | 화물·용달·짐 운송 |
| FREERUN | 퀵·심부름 |
| FREESHUTTLE | 농어촌형 호출 셔틀 |

서비스 종류가 달라도 아래 기능은 공통입니다.

- 로그인과 인증
- 호출 생성
- 가까운 기사 찾기
- 수락·거절 처리
- 실시간 상태 공유
- 운영자 화면

모양은 여러 서비스지만 뼈대는 하나입니다.

---

## 왜 앱이 3개인가요?

이동 서비스는 역할이 세 가지로 나뉩니다.

```
고객   →  호출을 만드는 사람
기사   →  호출을 받고 수행하는 사람
운영자 →  전체 상황을 보고 관리하는 사람
```

이웃콜은 이 세 역할에 맞게 화면을 나눴습니다.
실제로는 하나의 배차 시스템을 각자의 역할에서 보는 구조입니다.

---

## 실제로 어떻게 돌아가나요?

```
1. 고객이 앱에서 호출을 만든다
        ↓
2. 시스템이 주변 온라인 기사를 찾는다
        ↓
3. 기사 앱으로 배차 요청이 간다
        ↓
4. 기사가 수락하거나 거절한다
        ↓
5. 결과가 고객·운영자 화면에 실시간으로 반영된다
```

이웃콜은 "예약 폼"이 아닙니다.
이 흐름 전체를 자동으로 처리하는 배차 플랫폼입니다.

---

## 지금 이 저장소는 어디까지 되어 있나요?

아이디어 문서만 있는 상태가 아닙니다.
직접 실행하고 흐름을 눈으로 확인할 수 있는 수준까지 만들어져 있습니다.

**지금 바로 할 수 있는 것:**
- 고객 앱에서 회원가입·로그인·호출 생성
- 기사 앱에서 로그인 후 온라인 상태 전환·배차 수락
- 관리자 앱에서 기사 상태·통계 확인
- 세 화면에서 상태가 실시간으로 바뀌는 것 확인

**앞으로 더 발전할 것:**
- 실제 운영용 푸시·SMS 알림 연동
- 배포·모니터링 고도화
- 서비스별 정책·요금 체계 세분화
- 디자인·사용자 경험 개선

---

## 직접 체험하고 싶다면?

로컬 컴퓨터에서 실행해 볼 수 있습니다.
Node.js, Docker Desktop 설치가 필요합니다.

실행 방법은 [초보자 실행 가이드](./BEGINNER_GUIDE_KO.md)를 참고하세요.

처음 체험 순서:

1. 관리자 앱에서 개발용 토큰 발급
2. 기사 앱에서 로그인 후 온라인 상태 켜기
3. 고객 앱에서 호출 생성
4. 기사 앱에서 배차 수락
5. 관리자 앱에서 상태 변화 확인

이 과정을 한 번 보면 이웃콜이 왜 "단순한 앱 하나"가 아니라 "플랫폼"인지 바로 이해됩니다.

---

## 한 줄 정리

> 이웃콜은 수수료 없이 누구나 직접 운영할 수 있는 생활 이동 서비스용 오픈소스 배차 플랫폼입니다.
