# 이웃콜 비개발자 소개서

이 문서는 개발자가 아니어도 이웃콜이 무엇을 하려는 프로젝트인지, 왜 필요한지, 어떻게 체험하면 되는지 이해할 수 있도록 만든 짧은 소개서입니다.

## 1. 이웃콜은 무엇인가요

이웃콜은 택시, 대리, 화물, 퀵, 셔틀 같은 생활 이동 서비스를 운영할 수 있게 만드는 오픈소스 배차 플랫폼입니다.
쉽게 말하면 "고객이 호출하고, 기사가 수락하고, 운영자가 전체 상태를 보는 구조"를 직접 운영할 수 있게 만드는 기반입니다.

## 2. 왜 이런 프로젝트를 만들었나요

많은 플랫폼 서비스는 호출을 연결해 주는 대신 높은 수수료를 가져갑니다.
이웃콜은 배차 시스템 자체를 공개해서, 지역 서비스나 협동조합, 기사 중심 조직도 스스로 플랫폼을 운영할 수 있게 해 보자는 취지에서 시작했습니다.

이 프로젝트가 중요하게 보는 가치는 아래와 같습니다.

- 플랫폼 수수료 의존도를 줄이는 것
- 지역 기반 이동 서비스를 직접 운영할 수 있게 하는 것
- 서비스가 달라도 공통 기능은 함께 쓰게 만드는 것
- 특정 대기업 플랫폼에 묶이지 않는 구조를 만드는 것

## 3. 누가 관심 있게 볼 만한가요

이웃콜은 아래 같은 분들에게 특히 의미가 있습니다.

- 지역 기반 호출 서비스를 직접 운영하고 싶은 팀
- 기사 조합, 협동조합, 사회적 기업
- 공공형 교통 실험을 준비하는 조직
- 한국형 호출/배차 시스템 구조를 공부하고 싶은 사람

## 4. 이웃콜은 어떤 서비스를 생각하고 있나요

이웃콜은 하나의 앱만 만드는 프로젝트가 아니라, 여러 이동 서비스를 하나의 공통 코어로 묶는 구조를 지향합니다.

- `FREECAB`: 택시 호출
- `FREEDRIVE`: 대리운전
- `FREECARGO`: 화물/용달/짐 운송
- `FREERUN`: 퀵/심부름/다중 경유형 호출
- `FREESHUTTLE`: 농어촌형 호출 셔틀

핵심은 서비스 종류가 달라도 아래 기능은 공통이라는 점입니다.

- 로그인과 인증
- 호출 생성
- 가까운 기사 찾기
- 수락/거절 처리
- 실시간 상태 공유
- 운영자 화면

즉, 모양은 여러 서비스지만 뼈대는 하나입니다.

## 5. 왜 앱이 3개인가요

이웃콜은 역할이 다르기 때문에 화면도 나뉩니다.

- 고객 앱: 호출을 만드는 사람
- 기사 앱: 호출을 받고 수행하는 사람
- 관리자 앱: 전체 운영 상태를 확인하는 사람

처음에는 앱이 많아 보일 수 있지만, 실제로는 하나의 배차 코어를 서로 다른 역할에서 보는 구조라고 이해하면 쉽습니다.

## 6. 실제로는 어떻게 돌아가나요

가장 단순하게 보면 아래 흐름입니다.

1. 고객이 앱에서 호출을 만듭니다.
2. 시스템이 가까운 기사 후보를 찾습니다.
3. 기사 앱으로 배차 요청이 갑니다.
4. 기사가 수락하거나 거절합니다.
5. 결과가 고객과 관리자 화면에 실시간으로 보입니다.

즉, 이웃콜은 "예약 폼"이 아니라 "배차 흐름 전체"를 다루는 프로젝트입니다.

## 7. 첫 사용자는 무엇을 보면 되나요

처음 보는 분에게는 아래 순서를 추천합니다.

1. 이 문서로 취지를 이해합니다.
2. [서비스 개요 문서](./SERVICE_OVERVIEW_KO.md)로 구조를 봅니다.
3. [초보자 실행 가이드](./BEGINNER_GUIDE_KO.md)대로 직접 실행합니다.
4. 고객 앱, 기사 앱, 관리자 앱을 각각 열어 역할 차이를 봅니다.

## 8. 첫 체험은 어떻게 해보면 좋나요

로컬에서 실행한 뒤 아래 순서로 해보면 이해가 가장 빠릅니다.

1. 관리자 앱에서 개발용 토큰을 발급합니다.
2. 기사 앱에서 로그인하고 온라인 상태를 켭니다.
3. 고객 앱에서 호출을 만듭니다.
4. 기사 앱에서 배차를 수락합니다.
5. 관리자 앱에서 상태가 바뀌는지 확인합니다.

이 과정을 한 번 보면 "이웃콜이 왜 단순한 앱 하나가 아니라 플랫폼인지"가 바로 보입니다.

## 9. 지금 이 저장소는 어느 정도까지 되어 있나요

현재 저장소에는 아래가 포함되어 있습니다.

- 백엔드 API 코어
- 실시간 이벤트 구조
- 배차 큐
- 고객/기사/관리자 웹 앱
- 로컬에서 바로 띄워볼 수 있는 Docker 실행 흐름

즉, 아이디어 문서만 있는 상태가 아니라, 실제로 만져 보고 흐름을 확인할 수 있는 프로토타입 기반이 이미 있습니다.

## 10. 아직 남아 있는 일은 무엇인가요

아직은 Phase 0 단계이기 때문에, 아래 항목은 앞으로 더 발전할 수 있습니다.

- 실제 운영용 푸시/SMS 연동
- 운영 배포와 모니터링 고도화
- 서비스별 정책과 요금 체계 세분화
- 디자인과 사용자 경험 개선

## 11. 한 줄로 정리하면

이웃콜은 "생활 이동 서비스를 위한 공통 배차 코어를 오픈소스로 만들고, 누구나 직접 운영 가능한 형태로 제공하자"는 목표를 가진 프로젝트입니다.
